# Spotify Downloader

Downloads all songs from Spotify playlists as MP3s with full metadata and album art — **no Spotify API key required**. Compatible with [Navidrome](https://www.navidrome.org/) and any music server that reads `.m3u8` playlists.

## How it works

1. Reads `playlists/playlists.txt` for all playlist names + Spotify track URLs
2. Scrapes each public Spotify track page for title and artist
3. Searches YouTube for the best match via `yt-dlp`
4. Downloads as MP3 with embedded metadata (title, artist, album, cover art)
5. Builds a `.m3u8` playlist file per playlist when done

All output goes to `~/Music/SpotifyDownloads/<Playlist Name>/`.

## Usage

### Download everything
```bash
cd /path/to/this/repo
python3 download_all_ytdlp.py
```

### Download a single playlist by name
```bash
python3 download_all_ytdlp.py "Country"
```

### Override worker count
```bash
# Default is auto-detected (~1.5× your CPU count, max 20)
python3 download_all_ytdlp.py --workers 8
python3 download_all_ytdlp.py --workers 8 "Country"
```

### Run in the background (keeps going with lid closed)
```bash
# Start a detached tmux session from this repo folder
tmux new-session -d -s download \
  "cd $(pwd) && python3 -u download_all_ytdlp.py > ~/Music/SpotifyDownloads/master_log.txt 2>&1"

tmux attach -t download     # watch live progress
# Ctrl+B then D  →  detach without stopping
```

### Retry failed tracks (run after main script finishes)
```bash
python3 retry_failed.py
python3 retry_failed.py --workers 6   # optional override
```

### Fix tags for Navidrome (run after everything is downloaded)
```bash
python3 fix_tags.py
```
Looks up correct album, track number, year, genre, and cover art via MusicBrainz for any files with missing or bad tags. Safe to re-run — skips files that already have good tags. After it finishes, trigger a rescan in Navidrome.

---

## Worker count

The scripts auto-detect your CPU core count and pick a safe default (~1.5× cores, max 20 for the main script, max 12 for retry). More workers = faster, but too many triggers YouTube 429 rate-limit errors. The startup banner shows what's being used:

```
Workers   →  15  (CPU: 10  recommended: 15)
```

## playlists.txt format

```
Playlist Name
https://open.spotify.com/track/...
https://open.spotify.com/track/...

Next Playlist Name
https://open.spotify.com/track/...
...
```

- One playlist name as a plain-text header
- One Spotify track URL per line below it
- `spotify:local/` lines are automatically skipped
- Blank lines between playlists are fine

## Output structure

```
~/Music/SpotifyDownloads/
  Country/
    Artist - Song Title.mp3
    ...
    Country.m3u8          ← importable playlist file
    download_log.txt      ← OK / FAIL log per track
    .done_<track_id>      ← completion markers (enable safe restart)
  Oh shit/
    ...
  master_log.txt          ← full combined log (when run via tmux redirect)
  unfindable.txt          ← tracks that failed all retry attempts
```

## Navidrome

This project includes a `docker-compose.yml` that runs [Navidrome](https://www.navidrome.org/) — a self-hosted music streaming server — pointed at your downloaded library.

### Start the server
```bash
cd /path/to/this/repo
docker compose up -d
```

Then open **http://localhost:4533** in your browser and create your admin account. Navidrome will scan the library automatically.

### How playlists work

The `.m3u8` files use paths **relative to the library root** (e.g. `Country/Alabama - Dixieland Delight.mp3`), which is exactly what Navidrome expects. Once scanned, all playlists appear under the Playlists tab.

### What does NOT work in Navidrome playlists

- Absolute paths like `/Users/yourname/...`
- Paths outside the mounted `/music` folder
- External URLs

The playlists generated by this project are already formatted correctly.

### Tags Navidrome uses

| Tag | Used for |
|---|---|
| Title | Track name |
| Artist | Artist page |
| Album Artist | Album grouping |
| Album | Album page |
| Track number | Track ordering within album |
| Year | Sorting/display |
| Genre | Genre browsing |
| Cover art (embedded) | Album art everywhere |

Run `fix_tags.py` to ensure all these are populated correctly before scanning.

---

## Restarting safely

The downloader is fully idempotent. Each downloaded track gets a `.done_<id>` marker file. On restart, any track with a marker is skipped instantly — no re-scraping, no re-downloading. Just re-run the same command.

---

## Failure handling

| Type | Meaning | Fixed by retry script |
|---|---|---|
| `FAIL (scrape)` | Couldn't read the Spotify page | Retries the scrape |
| `FAIL (search)` | No YouTube result found | Retries with simplified query + 3 alternate results |
| `FAIL (download)` | yt-dlp failed to download | Retries with same + alternate URLs |

All failures log to `download_log.txt` per playlist. The retry script writes anything still unfindable after all attempts to `unfindable.txt`.
